# 修改内容说明 2026-06-09

本次修改围绕 OWASP Juice Shop 测试案例导入过程中暴露出的两个问题进行修复，目标是让系统能够稳定完成真实开源项目的上传、解压和预检。

## 修改一：修复 Windows 长路径导致 ZIP 解压失败

问题现象：

- 上传 `juice-shop-master.zip` 后，页面显示 `Internal Server Error`。
- 后端在解压 ZIP 时遇到较深层级路径，Windows 环境下触发路径长度限制，导致 `FileNotFoundError`。
- 结果是压缩包虽然部分解压成功，但导入流程中断，无法生成预检摘要。

修改内容：

- 在 `supplyguard/project_imports.py` 中增加 Windows 路径长度保护。
- 当解压目标路径过长时，跳过该文件，避免整个导入任务失败。
- TAR 解压流程也加入同样的路径长度保护。

修改效果：

- Juice Shop 这类包含深层目录结构的真实项目可以完成导入。
- 少量超长路径文件会被跳过，但不会影响整体预检流程。
- 页面不再因为该问题显示 500 错误。

## 修改二：修复导入项目被误判为系统运行目录

问题现象：

- Juice Shop 解压成功后，预检结果异常：
  - 总文件数能识别出来。
  - 但所有文件都被统计为 ignored。
  - 可扫描文件数为 0。
  - 依赖文件和 CI 文件无法正常识别。

根因：

- 上传项目会被解压到 `storage/imports/...`。
- 原有逻辑为了避免扫描系统自身运行数据，会跳过 `storage` 目录。
- 但这导致“被导入的测试项目”也被当成运行时目录跳过。

修改内容：

- 在 `supplyguard/project_imports.py` 中调整 `_is_runtime_workspace()` 判断逻辑。
- 如果当前扫描根目录本身位于导入工作区内，则不再把它当作需要忽略的系统运行目录。
- 仍保留对系统其他 `storage` 运行数据的保护，避免误扫平台自身缓存。

修改效果：

- Juice Shop 预检结果恢复正常。
- 当前验证结果：
  - 总文件：1375
  - 参与扫描：1109
  - 已忽略：133
  - 依赖文件：2
  - CI 文件：1
- 系统能够识别 `package.json`、`.gitlab-ci.yml` 和语言分布，为后续代码审计、依赖分析和 CI/CD 检测提供输入。

## 测试结论

修复后，OWASP Juice Shop 可以作为真实案例完成上传导入和预检，说明系统的项目导入、结构识别、依赖文件识别和 CI 文件识别流程已经具备基本可用性。
