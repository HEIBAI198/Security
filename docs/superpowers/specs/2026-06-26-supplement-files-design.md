# 补充文件链路设计

## 背景

安全平台当前在供应链可达性研判、CI/CD 构建链、产物可信门禁等模块中使用“上传证据”作为入口。这个文案容易让用户理解为一次性提交证据，而真实工作流中经常会出现初始导入时漏传文件、后续调查发现还需要构建日志、workflow、artifact 或 attestation 的情况。

本次改动把这些入口调整为“补充文件”，并确保补充文件进入与初始上传同等地位的处理链路：文件被当前模块识别、触发对应扫描或验证、结果回写当前 workspace，并参与后续图谱、报告和证据缺口判断。

## 目标

- 将供应链可达性研判、CI/CD 构建链、产物可信门禁中的“上传证据”统一改为“补充文件”。
- 补充的文件不能停留为孤立附件，必须进入对应模块已有的扫描或验证流程。
- 补充后的模块结果要回写到当前 workspace，与初始导入材料一起影响风险、图谱、报告和提示状态。
- 用户看到的按钮、空状态、toast、loading 文案要表达“补充后会重新处理”。

## 非目标

- 不新增全局文件中心。
- 不重做项目导入流程。
- 不改变后端存储模型中已有 evidence、workspace、module result 的结构，除非现有接口无法表达同级处理。
- 不把所有文件类型自动混投到所有模块；补充入口仍以当前模块为上下文。

## 方案

采用模块级补充文件链路。

### 供应链可达性研判

供应链可达性研判页的操作入口改为“补充文件”。当用户补充项目材料后，前端应触发与当前 workspace/import 关联的依赖与可达性扫描链路，刷新依赖风险、可达性证据、缺口提示和当前 workspace。

如果当前实现没有独立的“向现有项目追加源码文件”接口，本次前端先保持入口语义清晰：按钮文案、提示和重新扫描逻辑改为补充文件语义，并复用现有 `runDependencyAuditScan` / `runCodeAuditScan` 的 workspace 级重新处理能力。文件级追加接口可作为后续增强。

### CI/CD 构建链

CI/CD 模块入口改为“补充文件”。补充 workflow、构建配置或相关项目材料后，触发 `runCICDAuditScan` 的当前 workspace 级扫描，扫描结果合并回 workspace 的 `cicd_audit`、findings、modules 和图谱输入。

对于构建日志这类更接近日志印证的材料，界面文案应引导用户也可在日志模块补充；CI/CD 页面本身仍以 workflow、Action、runner、构建配置为主。

### 产物可信门禁

产物可信门禁入口改为“补充文件”。当用户选择 artifact 与 attestation 后，继续使用 `uploadArtifactTrustScan`，这与初始上传完全同级：后端保存上传文件、执行 digest/provenance/policy 校验，前端将结果写回 workspace 的 `artifact_trust` 并合并对应 findings/module summary。

如果用户还没有选择 artifact 或 attestation，界面保留待补充状态；两类文件齐备后执行同一验证逻辑。直接执行门禁的路径继续保留，用于已有 workspace/preset 材料。

## UI 文案

- 按钮：“上传证据”改为“补充文件”。
- 区块标题：“证据输入”可调整为“补充文件”或“文件输入”，强调补充后会重新验证。
- 成功 toast 使用“文件已处理”“补充文件已纳入研判”“产物可信验证完成”等表达。
- 缺材料提示使用“待补充”而不是“待上传证据”。

## 数据流

1. 用户在对应模块点击“补充文件”。
2. 用户选择文件或补齐模块所需输入。
3. 前端调用该模块已有扫描/验证 API。
4. API 返回模块结果。
5. 前端把模块结果合并进当前 workspace。
6. 依赖当前 workspace 的风险卡片、图谱、报告、证据缺口和 Agent 指引自然刷新。

## 错误处理

- 文件缺失：提示用户补齐当前模块必需文件，例如产物可信需要 artifact 与 attestation。
- 扫描失败：保持旧结果可见，显示失败 toast，不清空已有 workspace 数据。
- 超时：沿用现有超时提示，建议缩小范围或分模块重试。
- 非当前模块文件：通过文案引导到更合适的模块，不做静默丢弃。

## 测试

- 为前端文案和行为补充测试，覆盖按钮文案从“上传证据”变为“补充文件”。
- 覆盖产物可信补充 artifact/attestation 后仍调用 `uploadArtifactTrustScan` 并回写 workspace。
- 覆盖 CI/CD 和供应链模块重新扫描后更新当前 workspace 状态。
- 保留现有 API 类型与后端兼容性。

## 验收标准

- 供应链可达性研判、CI/CD 构建链、产物可信门禁不再出现面向用户的“上传证据”按钮。
- 用户补充文件后，当前模块进入 loading、调用对应处理链路，并把结果作为当前 workspace 的同级材料使用。
- 已有导出报告、图谱和证据缺口视图能读取补充后的模块结果。
- 补充失败不会破坏已有结果。
